Help / Information
What is a Certificate Authority?
A Certificate Authority (CA) is a trusted entity that issues digital certificates. These certificates verify the ownership of public keys and enable secure communications.
The certificate chain is verified from the end-entity certificate up to the root CA. Each intermediate CA must be properly configured and active.
Warning / Attention
Important
Revoking a certificate is irreversible. Make sure you have a valid replacement certificate before proceeding.
Your Root CA private key is extremely sensitive. Store it in a secure location and limit access to authorized personnel only.
Tips / Best Practices
Pro Tip
Set up automatic renewal notifications 30 days before certificate expiration to avoid service interruptions.
Use key sizes of at least 2048 bits for RSA certificates and consider 4096 bits for long-lived root CAs.
Danger / Critical
Danger Zone
Deleting a Certificate Authority will permanently remove all associated certificates and cannot be undone. Export backups before proceeding.
The Root CA private key has been compromised. Immediately revoke all issued certificates and generate a new CA.
